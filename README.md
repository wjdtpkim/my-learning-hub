# 내 학습허브 사이트

AI랑 함께 배우고 만든 것을 쌓는 나만의 사이트예요.
글(마크다운 `.md`)만 넣으면 사이트가 알아서 카드·목록을 만들어줘요. 레이아웃은 신경 안 써도 돼요.

> 💡 이 사이트를 도와줄 AI **학습메이트**는 1주차에 직접 만들어요. (지침 파일 4종: `SOUL.md`·`USER.md`·`AGENTS.md`·`CLAUDE.md` — 강의에서 만들기 프롬프트를 알려줘요)

## 어디에 뭘 넣나요?

모든 글은 `src/content/` 안에 넣어요. 폴더 = 글의 종류예요.

| 폴더 | 무슨 글 | 사이트에 보이나? |
|------|---------|------------------|
| `src/content/landing/` | **랜딩** (홈 화면 내 소개 글, `landing.md` 하나) | 홈 화면 위쪽 |
| `src/content/wiki/` | **학습위키** (로드맵·개념·사례·회고 다 여기) | 보임 (학습위키 메뉴) |
| `src/content/projects/` | 내 프로젝트 소개 | 보임 (프로젝트 메뉴) |
| `src/content/journal/` | 저널 (개인 학습일지) | **안 보임** (나만 봄) |

### 홈 화면(랜딩)은 어떻게?

홈 화면 위쪽은 `src/content/landing/landing.md`에 쓴 내 소개 글이 나와요. 그 아래는
프로젝트·학습위키 최근 글이 **자동으로** 딸려와요.

- **1주차에** 아이데이션·로드맵을 하고 나면, 그 대화 내용으로 `landing.md`를 채워요.
  (학습메이트한테 "이 대화로 내 랜딩페이지 글 만들어줘"라고 하면 편해요)
- 아직 안 만들었으면 홈은 "1주차에 여기를 채워요" 안내를 보여줘요. 지워도 사이트는 잘 나와요.

### 학습위키가 자동으로 정리되는 법

위키 글 맨 위에 `종류`와 `주차`만 적으면, 왼쪽 목록이 알아서 정리돼요.

- `종류: 로드맵` — 한 달 전체 지도. 맨 위 + 전자책 서문 (1주차에 하나)
- `종류: 개념` — 그 주에 배울 개념 (주차 시작)
- `종류: 사례` — 매일 쓰는 사례글 (하루 하나씩 쌓임)
- `종류: 회고` — 그 주 종합 (주차 끝, 발표용)
- `주차: 1` ~ `4` — 몇 주차 글인지

그래서 좌측 위키가 **로드맵 → 1주차(개념·사례·회고) → 2주차 …** 순으로 자라요.
그리고 이 순서 그대로 **전자책**이 만들어져요.

### 공통 규칙

- ★ **파일 이름은 영문·숫자·하이픈으로!** (예: `w1-case-day1.md`) 한글 파일 이름은 주소에서 깨져요. 제목은 안에 `title`로 한글로 쓰면 돼요.
- 각 폴더에 **견본 파일이 하나씩** 들어있어요(`공개: false`라 사이트엔 안 보여요). 새 글은 이 견본을 복사해서 내용만 바꾸면 제일 쉬워요. 보통은 학습메이트가 알아서 만들어주니 직접 만질 일은 적어요.
- 맨 위 `---`로 감싼 부분만 채우면 돼요. 그 아래는 자유롭게 마크다운으로.
- 어떤 글이든 안 보이게 하려면 맨 위에 `공개: false`를 넣으면 돼요.
- 본문에서 어려운 용어엔 자동으로 설명 풍선(툴팁)이 뜨고, 코드 상자엔 복사 버튼이 생겨요.

## 내 사이트 이름 바꾸기

`src/config/site.ts` 파일 하나만 열어서 이름·인사말·소개를 내 걸로 바꾸면 사이트 전체에 반영돼요.

## 사이트 미리 보기 / 배포

```bash
npm install     # 처음 한 번만 (필요한 부품 설치)
npm run dev      # 내 컴퓨터에서 미리보기 (http://localhost:4321)
npm run build    # 배포 전 이상 없는지 검사
```

배포는 GitHub에 올린 뒤 Vercel에 연결하면 자동으로 돼요. (강의에서 같이 해요)
