[GPTers 사례 — AI를 활용한 건강 데이터 분석 챗봇](https://www.gpters.org/health/post/health-data-analysis-chatbot-AYRCBY2pgoRVXVZ)

Garmin 스마트워치의 심박수·스트레스·수면 데이터를 자동으로 모아서, 카카오톡 챗봇으로 개인 맞춤 건강 인사이트를 알려주는 서비스를 만든 사례예요. 기술 스택이 많이 등장하지만(백엔드, 인프라 등), 여기서는 **AI가 데이터를 스스로 분석하는 방식**에 관한 핵심 개념만 정리해요.

## AI 에이전트

그냥 한 번 질문-답 하고 끝나는 챗봇이 아니라, 스스로 "어떤 데이터를 봐야 하지? 더 분석해야 하나?"를 판단하면서 여러 단계를 이어가는 AI를 에이전트라고 해요. 이 사례의 챗봇이 바로 이 방식이에요.

## ReAct 패턴

에이전트가 일하는 방식 중 하나예요. **추론(Reasoning)과 행동(Acting)**, 이 두 가지를 반복해요. "지난주 수면을 물어봤네 → 수면 데이터를 조회해야겠다 → 조회했더니 부족한데 → 스트레스 데이터도 봐야겠다"처럼, 스스로 생각하고 행동하고 그 결과를 보고 또 생각하는 방식이에요.

## 시계열 데이터

심박수·수면 기록처럼 **"언제 얼마였는지" 시간 순서가 중요한 데이터**를 시계열 데이터라고 해요.

## 실제로 겪은 문제와 해결

처음엔 AI에게 ReAct 방식으로 자유롭게 판단하게 맡겼더니, 두 가지 문제가 생겼어요:

- **무한 루프**: 같은 분석을 계속 반복하며 끝내지 못함
- **문맥 길이(컨텍스트) 한계**: 시계열 데이터를 한 번에 너무 많이 넣으면 AI가 제대로 다 못 읽음

그래서 AI가 자유롭게 판단하게 두는 대신, **"계획 세우기 → 도구 실행 → 분석 → 보고서" 4단계로 순서를 딱 정해서** 그 안에서만 움직이게 바꾸니 문제가 해결됐어요.

| 단어 | 쉬운 뜻 | 사례에서 어떻게 쓰였나 |
|------|---------|------------------------|

| ReAct 패턴 | 추론과 행동을 반복하며 스스로 다음 행동을 정하는 방식 | 초기 설계에 도입했다가 무한 루프 문제 발생 |
| 시계열 데이터 | 시간 순서가 중요한 데이터 | 심박수·스트레스·수면 기록 분석 |
