**한 줄 요약**: 코딩을 전혀 몰라도, GitHub 템플릿 → AI 학습메이트 세팅 → Vercel 배포까지 하루 만에 끝낼 수 있었어요.

**누구에게 도움될까**: 나처럼 비개발자인데 "AI랑 같이 뭔가 만들어보고 싶다" 하는 사람, 특히 학습 기록용 사이트를 실제로 인터넷에 띄워보고 싶은 사람.

## Before → After

- Before: 코딩 모름, GitHub·Vercel 둘 다 이름만 들어봤지 직접 써본 적 없음
- After: 실제 웹 주소(`my-learning-hub-seven.vercel.app`)로 접속되는 내 사이트 + 옆에서 같이 작업할 AI 학습메이트까지 완비

## 1단계 — AI 학습메이트 만들기

사이트 만들기 전에, 먼저 앞으로 4주 같이 갈 AI 파트너부터 정했어요. 파일 4개(`SOUL.md`·`USER.md`·`AGENTS.md`·`CLAUDE.md`)로 나눠서, AI한테 이렇게 부탁했어요:

> "학습메이트를 파일 4개로 만들 거야. SOUL.md(성격·말투), USER.md(나에 대해), AGENTS.md(작업 규칙), CLAUDE.md(입구 파일). 먼저 나한테 물어볼 것만 하나씩 인터뷰해줘."

AI가 이름 후보를 몇 개 제안해줘서 그중 하나로 골랐고("모닥"), 나를 어떻게 부를지, 내가 누구고 뭘 배우고 싶은지 3개만 인터뷰받고 답했더니 파일 4개가 바로 만들어졌어요.

💡 (원문엔 없어요 — 일반적인 방법입니다) 이렇게 "질문만 먼저 해달라"고 요청하는 걸 **인터뷰 방식**이라고 불러요. 한꺼번에 다 만들어달라고 하면 AI가 내 상황을 짐작해서 채우는데, 인터뷰로 하면 내 답으로 정확하게 채워져요.

## 2단계 — 로컬 미리보기 (막힘: Node.js 없음)

"내 컴퓨터에서 미리보기 띄워줘"라고 했더니, 이 컴퓨터엔 사이트를 실행시키는 **Node.js**가 없어서 안 됐어요. `brew`(자동 설치 도구)도 없어서, [nodejs.org](https://nodejs.org)에서 LTS 설치파일을 받아 직접 더블클릭으로 설치했어요.

- 설치 중 딱 한 번, 맥 로그인 비밀번호를 물어보는 창이 떴어요 — 이건 본인만 입력할 수 있는 부분이에요.
- 설치 끝나고 `node -v`로 확인한 뒤 `npm install` → `npm run dev`로 로컬 미리보기(`localhost:4321`)를 띄웠어요.

## 3단계 — GitHub에 올리기 (막힘: 업로드 버튼을 못 찾음)

GitHub 저장소를 새로 만들 때 "Add a README file"을 체크한 채로 만들어서, 원래 예상했던 "uploading an existing file" 링크가 안 보였어요. 대신 파일 목록 화면에서 **"Add file" → "Upload files"** 버튼으로 진행하니 됐어요.

- Finder에서 프로젝트 폴더 안 내용물 전체를 드래그해서 업로드 화면에 놓고 **Commit changes**.
- (스크린샷 자리 — GitHub "Add file → Upload files" 드롭다운 화면)

## 4단계 — Vercel로 실제 배포 (막힘 두 번)

Vercel 가입하고 프로젝트를 연결하는데 두 군데서 막혔어요:

1. **Team Name 빈칸**: 계정 만들 때 Team Name을 안 채워서 Continue 버튼이 계속 비활성화 상태였어요 → 아무 이름이나 입력하니 해결.
2. **저장소가 목록에 안 보임**: Vercel이 내 GitHub 저장소를 못 읽어서 Import Project 목록이 비어있었어요 → **"Install"** 버튼으로 GitHub 앱 권한을 먼저 연결해주니 그제야 저장소가 보였어요.

그 뒤 저장소 선택 → **Deploy** 누르니, Astro 프레임워크가 자동으로 인식돼서 몇 분 만에 실제 웹 주소가 생겼어요.

- (스크린샷 자리 — Vercel "Ready" 상태로 뜬 Production Deployment 화면)

## 배운 것 / 재사용 자산

- **AI 학습메이트 만들기 프롬프트 템플릿**: 위 1단계 문구를 그대로 복사해서 쓰면, 나만의 AI 파트너를 인터뷰 방식으로 빠르게 세팅할 수 있어요.
- **배포 체크리스트**:
  1. GitHub 저장소 만들기 (README 켜져 있으면 "Add file → Upload files"로)
  2. Vercel 가입 → GitHub 앱 Install → 저장소 Import → Deploy
  3. 이후 업데이트는 `src` 폴더만 다시 업로드하면 자동 재배포
- 막히는 지점이 대부분 "버튼 이름이 생각과 다른 곳에 있다" 종류였어요. 화면을 캡처해서 AI한테 보여주면 바로 다음 버튼을 짚어줘서, 혼자 헤매는 시간이 많이 줄었어요.
